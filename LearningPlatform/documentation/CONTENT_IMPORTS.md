# Content Imports (Bulk Data)


Besides the admin UI, you can load **tags, full courses (subject -> modules -> lessons -> tasks), extra modules on an existing course, and flashcard decks/cards** from import data files. Imports are **idempotent**: re-running updates matching records (slugs/order keys) instead of duplicating everything.

Add **`.js` files** under `scripts/imports/data/` (public) or `scripts/imports/data-private/` (local-only, gitignored) using CommonJS exports (`module.exports = ...`).
Each runner picks up every top-level `*.js` in matching public + private folders (non-recursive), **sorted by filename**.

## Why This Workflow Is Recommended

For large course generation, import scripts are the most reliable workflow in BrainStack:

- Fast for large content batches (much faster than manual block-by-block admin editing).
- Reliable for AI/LLM-generated course packages when exports follow the expected schema.
- In practical use, this has been more reliable for full-course ingestion than generating directly through the current in-app AI Agent flow.

## LLM Course Generation Notes

This pipeline is designed to support courses generated by external AI/LLM workflows:

- Most tested setup: **Claude Sonnet 4.5** (web UI) with a large custom prompt template.
- **Claude Haiku** and **ChatGPT/OpenAI** outputs also import successfully, but educational structure/quality usually needs more cleanup.
- A large structured prompt template is used to force output that matches BrainStack import shapes, making imports stable and predictable.
- I use a dedicated generation prompt: [creation_prompt.MD](./prompts/creation_prompt.MD).

## Requirements

A running database, `DATABASE_URL` (or `PAYLOAD_DATABASE_URL`) and `PAYLOAD_SECRET` in `.env`.

From the app directory (folder with `package.json`):

```bash
npm run content:import:all
```

Run order in full pipeline: **tags -> courses -> standalone modules -> flashcards**

| What | Folder | Export |
|------|--------|--------|
| **Tags** (Prisma) | `scripts/imports/data/tags/` | Array of `{ name, slug, main?: boolean }` |
| **Course tree** (Payload) | `scripts/imports/data/courses/` | One object `{ subject, course, modules }` per file |
| **Extra modules** (Payload) | `scripts/imports/data/modules/` | `{ courseSlug, module }` or `{ courseSlug, modules: [...] }` |
| **Flashcards** (Prisma) | `scripts/imports/data/flashcards/` | Either `[{ question, answer, tagSlugs? }]` or `{ deck?, cards: [...] }` |

Private/local-only equivalents are also scanned automatically:
`scripts/imports/data-private/tags/`, `scripts/imports/data-private/courses/`, `scripts/imports/data-private/modules/`, `scripts/imports/data-private/flashcards/`.

Tasks reference tags by **slug**; import tags before courses/flashcards that use them.
Course `level`: `BEGINNER` | `INTERMEDIATE` | `ADVANCED`.

## Shapes (Minimal Examples)

**`scripts/imports/data/tags/example.js`**

```javascript
module.exports = [
  { name: 'JavaScript', slug: 'javascript', main: true },
  { name: 'Basics', slug: 'basics' },
]
```

**`scripts/imports/data/courses/example.js`**

```javascript
module.exports = {
  subject: { name: 'Web', slug: 'web' },
  course: {
    title: 'JS Basics',
    slug: 'js-basics',
    description: 'Intro',
    level: 'BEGINNER',
    isPublished: false,
  },
  modules: [
    {
      title: 'Start',
      order: 1,
      lessons: [
        {
          title: 'Hello',
          order: 1,
          theoryBlocks: [
            { blockType: 'text', content: 'Hello world' },
            {
              blockType: 'image',
              image: '__IMPORT_PLACEHOLDER_IMAGE__',
              caption: 'Auto-resolved placeholder image (replace later in Admin -> Media if needed)',
              align: 'center',
              width: 'md',
            },
            {
              blockType: 'video',
              videoUrl: 'https://www.youtube.com/watch?v=jNQXAC9IVRw',
              title: 'Placeholder',
              caption: 'Swap URL anytime',
              aspectRatio: '16:9',
            },
          ],
          tasks: [
            {
              type: 'TRUE_FALSE',
              order: 1,
              prompt: 'JS runs in browsers.',
              correctAnswer: 'true',
              tagSlugs: ['javascript', 'basics'],
              points: 1,
            },
          ],
        },
      ],
    },
  ],
}
```

**`scripts/imports/data/modules/example.js`**

```javascript
module.exports = {
  courseSlug: 'js-basics',
  module: {
    title: 'More topics',
    order: 2,
    lessons: [{ title: 'Closures', order: 1, theoryBlocks: [], tasks: [] }],
  },
}
```

**`scripts/imports/data/flashcards/example.js`** (simple array form)

```javascript
module.exports = [
  { question: 'V8 is the JS engine in which browser?', answer: 'Chrome', tagSlugs: ['javascript'] },
]
```

**`scripts/imports/data/flashcards/example-with-deck.js`**

```javascript
module.exports = {
  deck: {
    slug: 'javascript-basics',
    name: 'JavaScript Basics',
    description: 'Core JS recall set',
    tagSlugs: ['javascript', 'basics'],
  },
  cards: [
    { question: 'What does `const` prevent?', answer: 'Reassignment of the binding', tagSlugs: ['javascript'] },
  ],
}
```

## Commands

| Script | Scope |
|--------|--------|
| `npm run content:import:all` | Full pipeline (recommended) |
| `npm run content:import:tags` | Tags only |
| `npm run content:import:course` | `scripts/imports/data/courses/` |
| `npm run content:import:modules` | `scripts/imports/data/modules/` |
| `npm run content:import:flashcards` | `scripts/imports/data/flashcards/` |

Flags: `--dry-run`, `--strict`, `--help` (runner dependent; strict validation is primarily used by course/module runners).

## Docker Workflow

```bash
docker compose --env-file .env exec app npm run content:import:all
```

Data files are baked in at image build unless you mount `scripts/imports`.
Rebuild or mount `scripts/imports` if you change data files; see `LOCAL_DEVELOPMENT.md` and the header comments in `docker-compose.yml`.

## Reliability and Re-runs

- Re-running imports **updates** existing Payload rows for the same natural keys.
- Flashcards dedupe by **question + deck + tag set** (order-insensitive).
- Flashcards **skip** when answer and tag set already match the existing card in that deck.
- If answer/tags changed for the matched flashcard, the row is updated.
- If tag slugs cannot all be resolved, import continues with a warning (import tags first for full linking).
- Orphan Payload rows after old deletes: `npm run payload:cleanup-orphans` (one-off maintenance).

## Important behavior details

- Course/module/lesson/task import defaults to published unless explicitly set `isPublished: false`.
- Course import upserts by slug; modules upsert by `(course, order)`; lessons by `(course, module, order)`; tasks by `(lesson, order, type)`.
- `CONTENT_IMPORT_SKIP_EXISTING_COURSES=true` skips updates/sync for courses that already exist by slug (recommended to preserve admin edits).
- Docker Compose defaults this flag to `1` for safety. Set `CONTENT_IMPORT_SKIP_EXISTING_COURSES=0` in `.env` only when you intentionally want imports to overwrite existing course trees.
- Theory `image` block value `__IMPORT_PLACEHOLDER_IMAGE__` resolves to a bundled placeholder media entry automatically.
- Course `coverImage` supports:
  - omitted: no change
  - `null`: clear cover
  - `__IMPORT_PLACEHOLDER_IMAGE__` or `__PLACEHOLDER__`: use bundled placeholder
  - `"filename.jpg"`: load from `scripts/imports/assets/`
  - numeric id: use existing media row

## Troubleshooting

**`Payload init failed: ... payload_locked_documents ... SET DATA TYPE uuid`**

This is caused by schema mismatch (not by the import writers themselves).

1. Align database once:

```bash
npm run payload:migrate
```

2. Import runners set `PAYLOAD_MIGRATING=true` to skip dev push during imports.

Then run:

```bash
npm run content:import:all
```

**`Invalid module "@payload-config"`** when running imports with plain `node`

Use npm scripts (or `tsx --tsconfig tsconfig.scripts.json ...`) so path aliases resolve:

```bash
npm run content:import:all
```
